## Philosophy

I enjoy rebuilding things from scratch—not because existing solutions aren't good enough, but because it's the fastest way I've found to truly understand them.

Whether it's implementing an expression parser in C, experimenting with reinforcement learning, or building backend services, I enjoy understanding the underlying ideas instead of only using existing libraries.

I'd rather go all-in on one problem until it's actually done than split my focus across five different things. There's always another rabbit hole worth exploring, but one at a time.

That doesn't mean I'd write a website in C, of course...

...although I'd probably think about it for five minutes.

In short tho...
I like going broad when solving a problem and going deep when understanding one.
